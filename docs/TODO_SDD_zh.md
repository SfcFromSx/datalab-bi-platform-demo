# DataLab - 优化待办事项列表 (TODO SDD)

本文件概述了 DataLab 平台的计划优化和未来工作，如软件设计文档 (SDD) 中所定义。

## 1. 智能体框架与通信

- [ ] **完整的 FSM 编排**: 增强 `ProxyAgent` 以实现完整的有限状态机 (FSM) 逻辑，用于在多个专业智能体 (SQL, Python, 图表等) 之间路由查询，而不仅仅是调用聊天智能体。
- [ ] **高级错误恢复**: 实现自校正循环，智能体可以分析执行错误并使用修改后的参数或代码进行重试。
- [ ] **流式智能体进度**: 确保所有智能体都支持通过 WebSocket 向前端发送细粒度的进度更新。
- [ ] **共享缓冲区持久化**: 添加将 `SharedBuffer` 持久化到数据库的选项，以支持长时间运行的多智能体任务。

## 2. 单元格运行时与上下文管理

- [ ] **工作区生命周期管理**: 为 `data/cell_runtime` 目录实现基于 TTL 或限制大小的清理工作器，以防止磁盘空间耗尽。
- [ ] **增强的变量追踪**: 改进 `variable_tracker` 以处理更复杂的 Python 模式 (例如 `globals()`, 动态属性访问和复杂的导入)。
- [ ] **增量 DAG 构建**: 优化 DAG 构建器，使其仅重新分析更改的单元格及其直接依赖项。
- [ ] **环境隔离**: 支持每个笔记本或每个单元格的虚拟环境 (使用 `venv` 或 `conda`) 以管理冲突的依赖项。

## 3. 知识模块

- [ ] **Map-Reduce 知识生成**: 完整实现 Map-Reduce 流水线，用于从历史脚本和架构中提取领域知识。
- [ ] **由粗到细的检索**: 在 `KnowledgeRetriever` 中实现加权评分机制 (词法 + 语义 + LLM)。
- [ ] **自动化专业术语提取**: 使用 LLM 从数据文档中自动识别和定义领域特定术语。

## 4. 执行引擎

- [ ] **生产环境沙箱**: 在生产环境中从简单的子进程过渡到更安全的隔离 (如 `gVisor`, `Firecracker` 或 Docker 容器)。
- [ ] **DuckDB 性能调优**: 为大规模内存 OLAP 操作优化 DuckDB 配置。
- [ ] **异步 SQL 执行**: 确保所有 SQL 执行都是完全异步的，以防止阻塞 FastAPI 的主事件循环。

## 5. 前端与 UI/UX

- [ ] **实时协作**: 为多用户笔记本编辑实现基于 CRDT 或操作转换 (OT) 的同步。
- [ ] **高级图表构建器**: 扩展 GUI 图表构建器，以支持复杂的 Vega-Lite 语法和更多 ECharts 类型。
- [ ] **单元格隐藏与分组**: 允许用户折叠或分组单元格，以实现更好的笔记本组织。
- [ ] **交互式数据概况**: 为每个表格输出添加可视化的数据概况选项卡 (直方图、相关矩阵等)。

## 6. 企业特性

- [ ] **完整的 RBAC 实现**: 在所有 API 端点中固化基于角色的访问控制逻辑。
- [ ] **审计日志可视化**: 为查看和搜索审计事件创建一个管理员仪表板。
- [ ] **SSO 集成**: 添加对 OIDC/SAML 提供商的支持，以实现企业级身份验证。
